Q1. LLMは主にどの学習方法で動いているか？
------------------------------------------------------------
A) 教師あり学習（ラベル付きデータから学習）
B) 教師なし学習（ラベルなしでパターンを発見）
C) 自己教師あり学習（生データから自ら学習タスクを作成）  ← 正解
D) 強化学習（報酬を最大化するように学習）

【解説】LLMは文中の単語を隠してモデルに穴埋めさせるなど、
データ自体が教師となる「自己教師あり学習」で動いている。
ラベルは不要で、インターネットほぼ全体を学習データとして使う。

Q2. バイブコーディングとはどのような開発スタイルか？
------------------------------------------------------------
A) エンジニアが一行ずつ手動でコードを書く従来の手法
B) 自然な言葉でAIに指示し、AIがコードを書いてくれる開発スタイル  ← 正解
C) クラウド上でコードを自動ホストする仕組み
D) テストだけをAIに任せる品質保証の手法

【解説】バイブコーディングは「構文ではなく、意図でコーディングする」考え方。
括弧やセミコロンに悩む必要はなく、欲しいものの「バイブ（雰囲気）」を
日常の言葉でAIに伝えるだけでコードが生成される。

Q3. 計画フェーズ（仕様作成・アーキテクチャ設計）に適したモデルはどれか？
------------------------------------------------------------
A) Sonnet / Codex（速くて確実なコード生成向け）
B) GPT-3.5（旧世代の軽量モデル）
C) Opus / GPT-5.5（深い推論が得意な高性能モデル）  ← 正解
D) LLaMA（オープンソースの小規模モデル）

【解説】ベストプラクティスは「計画にはOpus、実行にはSonnet」。
複雑な問題の分解や仕様の作成など深い推論が必要な作業にはOpus系、
複数ファイルの生成など素早い実行にはSonnet系を使い分ける。

Q4. Webアプリを構成する3つのファイルのうち、CSSの役割は何か？
------------------------------------------------------------
A) ページの骨格（見出し・ボタン・フォームの配置）を作る
B) クリックで動く機能（カート計算・ページ遷移）を担当する
C) 色・フォント・間隔・レイアウトなど見た目を担当する  ← 正解
D) サーバーとのデータ通信を管理する

【解説】Webの3ファイルの役割：
  HTML = 骨格（何が・どこにあるか）
  CSS  = 服と化粧（見た目・スタイル）  ← 正解
  JS   = 筋肉と脳（動く機能・ロジック）

Q5. 「PRD」とは何の略で、バイブコーディングにおいてどんな役割を果たすか？
------------------------------------------------------------
A) プログラム実行デバッグ（Program Runtime Debug）— バグを自動修正する仕組み
B) プロダクト要件定義書（Product Requirements Document）— AIへの設計図  ← 正解
C) プロジェクト報告書（Project Report Document）— 完成後に提出する文書
D) プロトタイプ実装設計（Prototype Requirement Design）— 画面の配色を決める書類

【解説】PRD（プロダクト要件定義書）は「AIへの設計図」。
目的・ユーザー・機能・ページ・データ・制約を明記することで、
AIが意図通りの高品質なアプリを作れるようになる。
「設計図が明確なほど、AIは高品質なものを作る」がセミナーの核心。
